# Flock Finder — Data Dictionary

Field-level reference for every published data artifact under `data/`.

> **Reminder:** all records are *suspected* Flock devices (OUI match only).
> Coordinates are published at full precision and are never modified. See
> [DATA_POLICY.md](DATA_POLICY.md).


## `data/flock_cameras.geojson`

A GeoJSON `FeatureCollection`. Each `Feature` is a `Point`.

### `geometry.coordinates`
`[longitude, latitude]` — full precision, exactly as reported by WiGLE.


### `properties`
| Field | Type | Description |
|-------|------|-------------|
| `netid` | string | BSSID / MAC address of the suspected device |
| `ssid` | string | WiFi network name as logged by WiGLE (may be empty/hidden) |
| `oui` | string | Matched OUI prefix, e.g. `70:C9:4E` |
| `match_confidence` | string | Always `"suspected"` |
| `channel` | number/string | WiFi channel (best effort) |
| `encryption` | string | Encryption type reported by WiGLE |
| `firsttime` | string | First time WiGLE observed the network |
| `lasttime` | string | Most recent observation (drives retention pruning) |
| `city` | string | Reverse-geocoded city (best effort) |
| `region` | string | State/region (best effort) |
| `country` | string | Country code (best effort) |
| `road` | string | Street name (best effort) |
| `postalcode` | string | Postal code (best effort) |

### Top-level `properties`
| Field | Description |
|-------|-------------|
| `generated` | ISO 8601 timestamp of the scan |
| `source` | `"WiGLE (wigle.net)"` |
| `match_confidence` | `"suspected"` |
| `total_cameras` | Number of features |


## `data/flock_cameras.csv`

Same records in flat form. Columns: `netid, ssid, trilat, trilong, oui_match,
channel, encryption, firsttime, lasttime, city, region, country, road,
postalcode`. Coordinates (`trilat`, `trilong`) are full precision.


## `data/scan_stats.json`

| Field | Description |
|-------|-------------|
| `scan_timestamp` | ISO 8601 time of the scan |
| `total_cameras` | Cumulative suspected devices |
| `new_this_scan` | Net-new records this run |
| `unique_ouis_found` | Distinct OUIs with ≥1 record |
| `ouis_queried` | OUIs queried this run |
| `api_requests` | WiGLE API calls made |
| `data_retention_days` | Retention window (730) |
| `cameras_by_oui` | Count per OUI |
| `cameras_by_region` | Count per region (top 50) |
| `cameras_by_country` | Count per country |

## `data/flock_ouis.csv` (canonical source of truth)

| Column | Description |
|--------|-------------|
| `oui` | 3-octet prefix (e.g. `70:c9:4e`) |
| `vendor_context` | Human note about the vendor/hardware |
| `detection_protocol` | How it was identified (e.g. WiFi 2.4 GHz) |
| `source` | Attribution (e.g. `@NitekryDPaul`) |
| `notes` | Free-form notes |

## `data/flock_ouis.json` (generated)

Machine-readable mirror of the CSV consumed by the frontend. Generated by
`python3 scripts/oui_metadata.py`. Contains `match_confidence: "suspected"`,
`total`, and an `ouis[]` array of `{oui, vendor_context, detection_protocol,
source, notes}`.

## `data/by_oui/<OUI_SLUG>.{geojson,csv}`

Per-OUI splits of the combined dataset (colons → underscores in the filename).
Same schema as the combined files.

## `data/scan_state.json` (local, gitignored)

Per-OUI incremental scan checkpoints. Not published; delete to force a full
rescan.
